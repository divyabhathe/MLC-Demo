# MLC-Demo

**MLC-Demo** is a development demo project that combines a **Next.js frontend chat interface** with a **FastAPI backend** to experiment with conversational access to student learning analytics.

The system allows users to chat with an AI assistant that can generate **read-only SQL queries** to retrieve context from a database and produce responses using an **LLM-powered backend**.

This project is intended for **development and experimentation**, not production deployment.

---

# Features

### Next.js Chat UI
- Simple web interface for interacting with the assistant.

### FastAPI Backend
- Accepts chat messages through the `/chat` API endpoint.
- Generates SQL queries using an LLM-based SQL agent.
- Retrieves contextual data from a database.
- Sends enriched prompts to an LLM to generate responses.

### SQL Safety Layer
- Restricts queries to **read-only operations**.
- Only allows queries starting with `SELECT` or `WITH`.

---

# System Flow

The system processes a user query through several backend components before returning a response to the frontend.

1. **User Prompt**
   - A user sends a message through the chat interface on the frontend.

2. **SQL Generation**
   - The backend passes the prompt to the **SQL agent**, which uses an LLM to generate a **read-only SQL query**.

3. **Database Query**
   - The backend executes the generated SQL query on the connected database using the `run_query()` function in `backend/database.py`.

4. **Context Construction**
   - The results from the database query are combined with the original user prompt to create a structured prompt.

5. **LLM Response Generation**
   - The backend sends this prompt to the LLM through `ask_zotgpt()` in `backend/zotgpt_client.py`.

6. **Response Delivery**
   - The LLM generates a **natural language response** based on the retrieved data.
   - The backend returns this response to the frontend chat interface.

# Privacy and AI Disclaimer

Responses generated by this system are **AI-generated** and may not always be perfectly accurate. The system is intended for **demonstration and development purposes only**.

To protect student privacy and comply with educational data regulations:

- The system is designed **not to expose personally identifiable student information**.
- Responses should **not include names or identifying information about other students**.
- The backend only retrieves **aggregated or contextual data** needed to answer a query.

This project is designed with awareness of major privacy regulations including:

- **FERPA (Family Educational Rights and Privacy Act)**
- **CCPA (California Consumer Privacy Act)**

This system ensures that:

- sensitive student data is properly protected
- database access controls are enforced
- AI-generated responses do not reveal confidential information
- institutional data policies are followed when connecting to real student datasets

```bash
git clone https://github.com/divyabhathe/MLC-Demo.git
cd MLC-Demo
```

---

# Running the Project

The **frontend and backend must run in separate terminals.**

---

# Backend Setup (Terminal 1)

Navigate to the backend directory:

```bash
cd ~/MLC/MLC-Demo/backend
```

Start the FastAPI server:

```bash
python3 -m uvicorn server:app --reload --port 8000
```

If successful, the backend will be available at:

```
http://localhost:8000
```

You should see a JSON status response when visiting the root endpoint.

---

# Frontend Setup (Terminal 2)

From the project root:

```bash
cd ~/MLC/MLC-Demo
npm run dev
```

The frontend will start at:

```
http://localhost:3000
```

The frontend (port **3000**) communicates with the backend (port **8000**).

---

# Configuration

## Database

Database access is implemented in:

```
backend/database.py
```

The project expects a function:

```python
run_query(sql: str)
```

This function should execute a query on a **PostgreSQL-compatible database** and return results as a string.

---

## LLM Client

LLM interactions are handled by:

```
backend/zotgpt_client.py
```

You must configure:

- your LLM API endpoint
- authentication keys
- environment variables for credentials
---

# Development Notes

### SQL Safety

The SQL agent in:

```
backend/sql_agent.py
```

filters queries to prevent destructive operations.

Current restrictions include:

- Only `SELECT` or `WITH` queries allowed
- Destructive SQL keywords are blocked

---

### Running Uvicorn Correctly

If you encounter import errors when starting the backend, ensure you are running the command **from the backend directory**.

Example working command:

```bash
cd backend
python3 -m uvicorn server:app --reload --port 8000
```
